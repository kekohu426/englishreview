# 自动化测试与开发循环系统

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                   自动化测试调度器                            │
│              auto_test_loop.js (每30分钟)                    │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ↓
         ┌───────────────────────┐
         │   执行测试用例          │
         │   - 简单词汇测试        │
         │   - Unit4完整测试       │
         └───────────┬───────────┘
                     │
                     ↓
         ┌───────────────────────┐
         │   调用后端API          │
         │   POST /api/generate   │
         └───────────┬───────────┘
                     │
                     ↓
         ┌───────────────────────┐
         │   分析测试结果          │
         │   - 题型覆盖检查        │
         │   - 性能分析            │
         │   - 错误分析            │
         └───────────┬───────────┘
                     │
                     ↓
         ┌───────────────────────┐
         │   生成测试报告          │
         │   - JSON详细报告        │
         │   - TXT摘要报告         │
         │   - 优化建议列表        │
         └───────────┬───────────┘
                     │
                     ↓
         ┌───────────────────────┐
         │   开发团队查看          │
         │   根据建议优化代码      │
         └───────────────────────┘
```

## 使用方法

### 1. 启动自动化测试循环

```bash
cd D:\dev\english-review-app

# 终端1：启动后端服务器（保持运行）
npm start

# 终端2：启动自动化测试（保持运行）
node auto_test_loop.js
```

### 2. 系统会自动：

- ✅ 立即执行第一轮测试
- ✅ 每30分钟自动执行一次
- ✅ 生成详细报告到 `test-reports/` 目录
- ✅ 分析问题并给出优化建议

### 3. 查看测试报告

```bash
# 查看最新报告
cd test-reports
ls -lt | head -5  # 最新的5个报告

# 查看摘要
cat run-2026-06-12T12-30-00-000Z.txt

# 查看详细JSON
cat run-2026-06-12T12-30-00-000Z.json
```

## 报告格式

### 摘要报告 (.txt)

```
# 自动化测试报告

运行ID: run-2026-06-12T12-30-00-000Z
时间: 2026/6/12 20:30:00

## 总览

- 总测试数: 2
- ✅ 通过: 1
- ❌ 失败: 1
- 🚨 错误: 0
- 成功率: 50.0%

## 详细结果

### 1. 简单词汇测试
- 状态: PASS
- 耗时: 15.3秒
- 题目数: 60
- 题型覆盖: 12/12

### 2. Unit4完整测试
- 状态: FAIL
- 耗时: 45.8秒
- 错误: 题型覆盖不完整

## 🔧 优化建议

### 1. [HIGH] 缺少题型: dialogue_complete
- 建议: 检查fallback逻辑是否正常工作
- 文件: backend/generators/fallback.js
- 操作: 确保兜底逻辑正确生成缺失题型
```

### JSON报告 (.json)

包含完整的测试数据，供程序化分析：
- 每个测试用例的详细结果
- 题型统计
- 性能数据
- 错误信息
- 优化建议列表

## 优化建议类型

系统会自动识别以下问题并给出建议：

1. **API速率限制 (429错误)**
   - 增加批次延迟
   - 降低并发数

2. **API超时 (504错误)**
   - 精简prompt
   - 优化请求内容

3. **JSON解析失败**
   - 改进cleanJsonResponse函数
   - 增强错误处理

4. **题型覆盖不完整**
   - 检查fallback逻辑
   - 优化Stage1 prompt

5. **性能慢（>60秒）**
   - 增加并发
   - 优化生成策略

## 开发优化工作流

### 步骤1：查看最新报告

```bash
cd test-reports
cat $(ls -t *.txt | head -1)
```

### 步骤2：根据优化建议修改代码

例如报告显示：
```
[HIGH] API速率限制
文件: backend/config.js
操作: 将 stage2.delayMs 从 2000 增加到 3000
```

执行修改：
```javascript
// backend/config.js
stage2: {
  batchSize: 1,
  delayMs: 3000,  // 从2000改为3000
}
```

### 步骤3：重启服务器

```bash
# 停止当前服务器 (Ctrl+C)
# 重新启动
npm start
```

### 步骤4：等待下一轮自动测试

- 无需手动触发
- 系统会在下一个30分钟周期自动测试
- 查看新报告验证改进效果

## 手动触发测试

如果不想等30分钟，可以手动运行：

```bash
# 单次测试（不循环）
node test_ultra_mini.js

# 或完整测试
node test.js
```

```bash
# 在运行 auto_test_loop.js 的终端按 Ctrl+C
```

## 注意事项

1. **API速率限制**：系统会自动在测试间延迟3秒，避免触发速率限制
2. **服务器必须运行**：确保 `npm start` 在后台持续运行
3. **报告清理**：定期清理 `test-reports/` 目录的旧报告
4. **错误恢复**：如果某次测试失败，系统会继续下一轮，不会停止
