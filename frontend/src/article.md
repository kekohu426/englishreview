# 我用 Figma 做了一个小学英语复习 App，才发现什么叫真正的「产品思维」

---

> 这不是一篇软广。这是一个普通人用 AI 工具搭产品的真实记录，以及一些关于「设计工具和产品思维」的碎碎念。

---

## 起因：一个群里的日常需求

我孩子班上的英语老师，每次复习前都会在家长群里发一段文字：

> "这周复习 Unit 3-4，重点是动物单词、颜色形容词，还有 'What is it?' 的问答句型，家长监督孩子多读多听。"

然后，家长们就各显神通——有人打印卷子，有人找 APP，有人啥也不做。

我就想：**能不能让 AI 直接读这段需求，自动生成一套互动题目，孩子拿手机就能做？**

这个想法让我开始动手，也让我第一次认真思考：**一个工具「好用」和「有产品思维」到底差在哪里。**

---

## 我选了 Figma Make，原因很朴素

我不是前端工程师。我会写一点代码，但不擅长从零搭工程。

之前试过几个「AI 生成网页」的工具，生成出来的东西基本上是：能看，不能用，改一下就崩。

Figma Make 不太一样。它生成的是真实运行的 React 项目，组件之间有逻辑，状态管理是通的，我可以一边看预览、一边改代码，改了马上生效。

更重要的是：**它让我把「设计稿」和「跑起来的产品」放在同一个地方思考。** 不用来回切软件，不用「设计是一套、实现是另一套」。

这件事听起来很小，但对一个想做产品验证的人来说，心理摩擦少了很多。

---

## 我做了什么：12 种题型，一套完整的儿童英语复习系统

我把整个 App 的核心拆成三层：

**第一层：首页（HomeScreen）**
老师发的复习需求被解析成若干「模块」，每个模块有颜色标识、技能标签（🎧听 / 🎤说 / 👁️读 / ✍️写），孩子一眼看到要复习什么。

**第二层：题目流（LessonScreen）**
每个模块包含多道题，答对自动推进，顶部有进度条，底部有导航按钮。答错了有解析，可以重试。

**第三层：12 种题型**
这是我花时间最多的地方。每种题型都有独立的交互逻辑和视觉风格，包括：

| 题型 | 核心体验 |
|------|---------|
| 🎧 听音选图 | 点喇叭 → 从三张图里选正确的单词 |
| 🎧 听音选答 | 听问题 → 选择最合适的英文回答 |
| ⚖️ 听句判断 | 听一句话 → 大按钮判断「对 / 错」 |
| 🎤 跟读配音 | 倒计时 → 录音 → AI 打分（78-99分） |
| ✍️ 字母描写 | 听发音 → 在有规则线的画板上描写字母 |
| 🔤 词语排序 | 点击单词卡片 → 拼成正确句子 |
| ✏️ 句子填空 | 句子中有空白 → 点选词填入 |
| 👁️ 看词认图 | 看大号英文单词 → 选对应图片 |
| 🔡 单词拼写 | 听发音 → 点字母卡一个个拼出单词 |
| 🌏 中英互译 | 看中文选英文，或看英文选中文 |
| 💬 对话补全 | 看对话气泡 → 选出正确的回答补全对话 |
| 🎲 综合问答 | 以上的综合混搭题 |

## 聊一聊「产品思维」这件事

做这个 Demo 的过程中，我反复在想一个问题：**同样是「AI 帮你生成界面」，为什么有些工具用完感觉在玩具，有些用完感觉在做产品？**

我总结下来，差距在三个地方：

### 1. 从需求出发，而不是从功能出发

Figma Make 让我先想清楚「孩子需要什么样的题型体验」，再去生成对应的组件，而不是先给我一堆通用 UI 组件让我自己拼。

这种「以用户场景为中心」的思路，其实就是产品经理天天挂在嘴边的那句话。

### 2. 设计即交互，不是装饰

在做「判断对错」题型的时候，我刻意把按钮做得很大、颜色很鲜明（绿色 vs 红色），是因为目标用户是 6-9 岁的小孩——他们需要一眼知道「这个大绿块是答「对」的」，而不是读按钮上的字。

**设计决策背后是用户认知，不是「好不好看」。**

### 3. 状态流转是产品核心，不是技术细节

整个 App 最复杂的部分不是某一道题的 UI，而是：答对了怎么进入下一题？答错了给不给重试？模块完成后保存什么数据？跨会话怎么记住进度？

Figma Make 让我在一个地方同时考虑「界面长什么样」和「状态怎么流动」，这两件事一旦分开就会产生大量「设计说不清楚、工程做不明白」的损耗。

## 最后

这个 Demo 还很粗糙：发音打分是模拟的，AI 解析需求的部分还没做，词库只有 18 个单词。

但它让我相信一件事：**用对工具，一个人也可以在几天内把一个真实的教育产品想法跑通。**

Figma 一直在做的事，不只是「设计工具」，而是让「想法到产品」的链路变短。这次的 Make 功能，是我见过目前最接近这个目标的一步。

如果你也有一个想验证的产品想法，可以试试。
